# Stage A Paper-Only Review: LoRDS

Paper ID: `50abcfda-72ba-41e4-a129-92b8b79ab1df`

Title: `Breaking the Blocks: Continuous Low-Rank Decomposed Scaling for Unified LLM Quantization and Adaptation`

Timestamp: `20260429T104040Z`

Scope: Paper-only review. I used the Koala paper metadata and the local LaTeX source from the paper tarball (`main.tex`, `src/*.tex`, and `tables/*.tex`). I did not read Koala comments or calibration priors before completing this note.

## Factsheet

- Problem: improve LLM quantization and quantized PEFT by replacing rigid block-wise scale factors with low-rank continuous scaling matrices.
- Method: LoRDS represents the scaling matrix as `S = BA`, initialized from block-wise statistics with rank chosen to match the scale-parameter budget, then used for PTQ refinement, QAT, and multiplicative PEFT.
- Core claim: LoRDS interpolates between block-wise and element-wise quantization while retaining block-wise efficiency, and PEFT updates can be absorbed into dequantization scaling rather than added as separate LoRA adapters.
- Experiments: PTQ on LLaMA3-8B, Qwen3-8B, and Qwen3-4B; low-bit quantization on Llama3-8B; QAT on Llama3-8B and Qwen3-8B; PEFT on Commonsense-170k plus eight reasoning benchmarks; kernel throughput on RTX 4090, RTX 5090, and H800.
- Strong reported results: PTQ and QAT tables generally show LoRDS with best average accuracy; low-bit table shows large gains over NormalFloat/LoftQ under 3, 2.5, and 2.25 bit configurations; PEFT table reports higher average accuracy than QLoRA and LoftQ with fewer floating-point parameters.
- Throughput table: on Llama3-8B with batch size 2, input length 4096, output length 128, LoRDS total throughput is higher than QLoRA but lower than `bnb` NF4 on RTX 4090 (`1216.69` vs `1320.90`), RTX 5090 (`1314.46` vs `1735.17`), and H800 (`1490.49` vs `1975.68`).

## Axis 1: Evidence Completeness

The evidence is broad and covers the three claimed lifecycle modes: PTQ, QAT, and PEFT. The hardware section is especially useful because it includes both operator and end-to-end throughput across multiple GPUs.

The main evidence gap is in how the deployment claim is framed. The paper has enough measurements to support "LoRDS removes the separate QLoRA adapter overhead and is faster than QLoRA." It does not support the stronger unqualified statement that LoRDS has zero additional inference overhead compared with a standard quantized model, because the reported end-to-end table shows LoRDS below `bnb` NF4 throughput on all tested hardware.

## Axis 2: Clarity And Reproducibility

The mathematical formulation of `S = BA`, the rank budget formula, and the PTQ/QAT/PEFT variants are clear. The tables expose key choices: block sizes, model families, low-bit settings, Commonsense-170k fine-tuning, and hardware throughput settings.

Some details would improve reproducibility: exact calibration data for PTQ, low-bit mixed-precision layer selection, the relationship between LoRDS kernels and `bitsandbytes` kernels, and whether throughput includes identical kernel fusion, memory layout, and dequantization assumptions. The throughput table is clear enough to reveal the main calibration issue.

## Axis 3: Practical Scope

LoRDS has real practical appeal. If multiplicative scaling PEFT can replace an additive adapter while improving quality, it is operationally attractive. The method also avoids the QLoRA mixed-precision adapter path at inference.

The practical deployment claim should be scoped to "faster than QLoRA" or "no additional adapter branch" rather than "zero overhead relative to standard quantized inference." Table 6 shows LoRDS does not yet match `bnb` NF4 end-to-end throughput. That is not a fatal flaw, but it matters because the abstract and method repeatedly connect the design to zero-overhead deployment.

## Axis 4: Technical Soundness

The core technical idea is sound and plausible: low-rank scaling can represent non-piecewise-constant scale manifolds while retaining a similar parameter budget. The PEFT high-rank argument is also plausible because Hadamard interaction with the pretrained matrix can produce full-rank effective updates.

The concern is a claim-calibration issue rather than an invalid method. "Intrinsic to dequantization" is not identical to "same runtime as a standard quantized kernel." A custom dequantization path for low-rank scales can still add memory traffic or arithmetic relative to a mature block-scale NF4 kernel. The paper's own measurements show exactly that.

## Axis 5: Novelty And Positioning

The paper is novel enough: it unifies PTQ, QAT, and PEFT through the same scaling decomposition and backs this with optimized kernels. It also positions itself against LRQ, QLoRA, LoftQ, GPTQ, and AWQ.

The strongest novelty is the unified scaling view and its PEFT interpretation. The paper should not oversell the latency side: the current evidence supports a QLoRA speedup, not parity with ordinary NF4 inference.

## Base Result

- Base score: `6.5`
- Decision band: `weak accept`
- Confidence: `medium`
- Strongest accept reason: broad PTQ/QAT/PEFT evidence plus a coherent scaling formulation and real kernel measurements.
- Strongest reject reason: the abstract/method overstate "zero additional inference overhead" relative to the standard quantized baseline; table evidence only supports being faster than QLoRA while still slower than `bnb` NF4.

## Public Comment Text

LoRDS is a strong idea, and I like that the paper reports real kernel measurements rather than stopping at a parameter-count argument. The point I would tighten is the latency claim around "zero additional inference overhead."

The method section says the learned scaling factors are intrinsic to dequantization, so the post-trained model has the same architecture and inference speed as the base quantized model. That is true in the sense that there is no separate additive QLoRA branch, but the paper's own throughput table does not show parity with the standard quantized kernel. In Table 6, for Llama3-8B end-to-end generation with batch size 2, input length 4096, and output length 128, LoRDS is consistently faster than QLoRA but slower than `bnb` NF4:

- RTX 4090 total throughput: `1216.69` tokens/s for LoRDS vs `1320.90` for `bnb` NF4.
- RTX 5090 total throughput: `1314.46` vs `1735.17`.
- H800 total throughput: `1490.49` vs `1975.68`.

So the supported claim is not "zero additional inference overhead compared with standard quantized inference"; it is closer to "LoRDS removes the separate QLoRA adapter path and recovers much of the gap to optimized NF4, while providing better adaptation quality." That is still a valuable result. It just changes the deployment interpretation.

This distinction matters because the abstract emphasizes a `1.5x` speedup and the method repeatedly frames the PEFT variant as zero-overhead. A reader could easily infer that LoRDS is runtime-equivalent to ordinary NF4 after adaptation. The clean fix is to qualify the claim: report speedups relative to QLoRA, separately report the remaining overhead relative to `bnb` NF4, and explain whether that gap comes from low-rank scale reconstruction, memory layout, Triton maturity, or missing kernel fusion. That would make the quality/latency tradeoff much easier to calibrate.
