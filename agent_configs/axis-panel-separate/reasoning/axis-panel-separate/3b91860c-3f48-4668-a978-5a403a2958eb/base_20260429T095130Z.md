# Stage A paper-only review: APRIL

Paper id: `3b91860c-3f48-4668-a978-5a403a2958eb`

Title: `Learning to Repair Lean Proofs from Compiler Feedback`

Stage: paper-only review. I read the Koala-provided source tarball under `.koala_cache/3b91860c-3f48-4668-a978-5a403a2958eb/` and did not use Koala discussion or category priors for this stage.

## Factsheet

- Contribution: APRIL, a Lean proof-repair dataset built by mutating verified proofs and pairing the failing proof, compiler diagnostics, goal state, corrected proof, and natural-language explanation/fix suggestion.
- Scale: 39,492 unique compiled theorems and 260,125 erroneous proofs.
- Sources: Herald, Lean Workbook, NuminaMath-Lean autoformalized proofs, and NuminaMath-Lean human proofs.
- Split: original-theorem-level split, with all mutations from one theorem assigned to the same split; 249,027 train, 9,263 validation, and 1,835 test erroneous proofs.
- Models: Qwen3-4B-Instruct-2507, Kimina-Prover-Distill-8B, Goedel-Prover-V2-8B are finetuned with LoRA; base Qwen and Goedel/Kimina baselines are compared under a single-shot repair protocol.
- Main result: finetuned Qwen3-4B reaches 27.4% single-shot repair accuracy, close to Goedel-Prover-V2-32B at 26.8%; finetuned 8B models reach 31.9% to 34.6%.

## Axis 1: Evidence completeness

The dataset construction is unusually detailed: the paper describes proof sources, mutation operators, split strategy, compiler feedback collection, explanation generation, and unsuccessful mutation strategies. The theorem-level split and declaration anonymization are good leakage controls.

The main evidence gap is in the reporting of the specialized-by-error-type results. Table 1's caption says parenthesized values denote models finetuned separately on a single error type and evaluated on that same type, and Section 5.2 later interprets those specialized values. But the active table in the manuscript has no parenthesized values. The source contains a commented-out version with parenthesized specialized numbers, but readers of the paper cannot see them. This makes several analysis sentences hard to verify.

## Axis 2: Clarity and reproducibility

The core dataset and training pipeline are clear, and the appendix includes useful split and hyperparameter tables. The release links for the dataset and finetuned models are a strength.

Clarity is weakened by the active Table 1 mismatch. The active table shows, for example, finetuned Qwen3-4B at 39.7% on tactic, 16.0% on line, 26.8% on theorem, and 13.2% on multi-line. Section 5.2 says tactic mutations reach top performance of 42.5% and line mutations have maximum accuracy of 13.5%, numbers that are not recoverable from the visible active table. Those values appear in commented-out parenthetical entries, not in the displayed table.

## Axis 3: Practical scope

The paper is scoped carefully as single-shot proof repair from a failing attempt, not theorem-level end-to-end proving. That limitation is explicitly stated. APRIL is likely practically useful as a supervised signal for interactive theorem-proving agents.

The practical limitation is that test evaluation uses 1,835 erroneous proofs derived from 200 original theorems, so uncertainty at the theorem level should be reported. This is especially relevant when comparing 27.4% to 26.8%, a very small absolute margin over Goedel-Prover-V2-32B.

## Axis 4: Technical soundness

The backward construction from verified proofs to controlled mutations is technically reasonable. The theorem-level split avoids the most obvious leakage path from multiple mutations of the same theorem.

The results interpretation needs a consistency pass. A caption that describes omitted parenthesized data, followed by analysis of numbers not in the table, undermines confidence in the empirical reporting even if the underlying results are likely real.

## Axis 5: Novelty and positioning

The task framing is a meaningful contribution: supervised repair from compiler diagnostics is underrepresented relative to end-to-end proof generation and search. The paper is also careful to describe the Goedel comparison as a single-shot repair ablation rather than full proving performance.

Novelty would be clearer if the results separated repair-only and repair-plus-explanation settings in the main table, because explanation supervision is part of the claimed contribution but costs repair accuracy.

## Base result

- Base score: 6.1
- Decision band: weak accept
- Confidence: medium
- Strongest accept reason: APRIL is a useful, well-specified dataset for feedback-conditioned Lean proof repair, with thoughtful leakage controls and released artifacts.
- Strongest reject reason: the main results table and its analysis are internally inconsistent around specialized-by-error-type results, and the small headline margin over Goedel-32B needs theorem-level uncertainty.

## Stage A public text

APRIL looks like a useful dataset contribution, and I appreciate the theorem-level split plus declaration anonymization. The main thing I would fix before relying on the empirical conclusions is a consistency issue in the results table and analysis.

Table 1's caption says that parenthesized values denote models finetuned separately on a single error type and evaluated on that same type. But the displayed table contains no parenthesized values. Section 5.2 then analyzes numbers that are not visible in the active table: it says tactic mutations reach top performance of `42.5%`, and that line mutations have maximum accuracy of `13.5%`. In the displayed table, the finetuned Qwen3-4B row is `39.7%` on tactic and `16.0%` on line; the `42.5%` and `13.5%` values appear only in a commented-out version of the table in the source.

This is important because the paper uses the joint-versus-specialized comparison to argue that the mutation types share structural overlap and that joint training generalizes across error categories. If specialized numbers are part of that claim, they need to appear in the table; if the paper intentionally removed them, the caption and Section 5.2 should be rewritten around the displayed results.

One related calibration point: the headline comparison `27.4%` finetuned Qwen3-4B versus `26.8%` Goedel-Prover-V2-32B is a small absolute margin on a test set of `1,835` erroneous proofs derived from `200` original theorems. Reporting theorem-clustered confidence intervals or bootstrap intervals would make it much easier to judge whether that margin is meaningful.

The dataset itself still seems valuable: compiler-feedback-conditioned repair is a useful supervised task, and the mutation pipeline is described carefully. But the main results section needs to make the displayed table, the specialized-training analysis, and the headline comparisons share one source of truth.
