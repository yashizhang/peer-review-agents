## Paper Factsheet

- Domains: d/NLP, d/Optimization.
- Main contribution: R2-Router extends LLM routing from selecting only a model to selecting a model and output-length budget, supported by R2-Bench, a dataset of multiple responses per query/model under different token budgets.
- Claimed novelty: existing routers treat each model as a single quality-cost point; R2-Router models quality-cost curves and searches over model-budget pairs.
- Main empirical setup: R2-Bench contains 30,968 queries from six benchmarks; each query is evaluated across 15 LLMs and 16 token budgets; responses are judged by Qwen3-80B-Instruct and annotated with actual token count.
- Cost setup: the paper says costs follow OpenRouter prices and Appendix B lists both input and output prices, retrieved in January 2026.
- Main results: R2-Router claims comparable quality at 4-5x lower cost, OOD improvement on MMLU-Pro STEM to non-STEM, robustness to embeddings and judge choice, and integration with UniRouter.
- Theory: Theorem 1/Optimization Dominance is a search-space superset guarantee: if the reactive router's predicted cost is in the feasible budget set, the reasoning search space can weakly dominate.
- Key caveat from paper source: Section 4.1 says responses are collected under budgets enforced by prompt and truncation, and annotated with actual token count; Appendix A reports compliance rather than perfect adherence, with small models especially weak at tight budgets.
- Artifact caveat: the submitted platform metadata exposes no GitHub URL; the tarball includes LaTeX/figures/tables but not raw R2-Bench traces or replay code for the headline frontier.

## Sub-Agent Outputs

### Evidence Completeness Agent

Axis score: 4/10
Accept/reject signal: weak reject
Confidence: medium

#### Strongest evidence

- The paper evaluates multiple baselines, reports five random seeds, and includes OOD, judge, embedding, predictor, interpolation, and prompt-ablation experiments.
- R2-Bench is a useful dataset concept because it records multiple model responses across token budgets rather than a single default response.

#### Main concerns

- The headline 4-5x lower-cost result is not replayable from the submitted artifact and depends on cost accounting choices not fully exposed in raw traces.
- The comparison mixes requested budget, actual token count, input-token price, output-token price, and truncation/compliance behavior; the paper does not provide enough per-query accounting to verify the Pareto frontier.
- The theorem supports only a superset optimum, not the learned router's realized quality-cost performance.

#### Missing checks that would change the decision

- A raw per-query trace table with model, requested budget, actual output length, input length, input and output cost, quality score, compliance, and chosen route.
- An oracle-vs-learned decomposition separating curve-space benefit from predictor error and cost-accounting effects.

### Clarity and Reproducibility Agent

Axis score: 4/10
Accept/reject signal: weak reject
Confidence: medium

#### What is clear

- The model-budget routing formulation, dataset construction, and high-level MLP predictor setup are understandable.
- The paper lists the LLM pool and per-token prices.

#### Reproducibility blockers

- No code or raw benchmark data are exposed through the platform metadata.
- January 2026 prices are time-sensitive, and the paper does not include a pinned manifest/raw trace sufficient to reconstruct the reported deferral curves.

#### Clarifying questions

- Are deferral curves computed with requested output budgets, actual generated output tokens, full input-plus-output API cost, or some hybrid? The text points to different pieces but does not expose the full accounting rule.

### Practical Scope Agent

Axis score: 4/10
Accept/reject signal: weak reject
Confidence: medium

#### Scope supported by evidence

- The basic paradigm is practically relevant: model routing should account for controllable output length and quality-cost curves.
- The paper includes an OOD split and dynamic-pool UniRouter integration.

#### Generalization / robustness / efficiency concerns

- Routing overhead is reported as under 400 ms and under 1% of generation time, but end-to-end latency/cost is not fully decomposed.
- Compliance is non-perfect, and low-budget cells mix naturally concise answers, truncated answers, and non-compliant attempts unless stratified.

#### Stress tests worth asking for

- Full API-cost replay under current/pinned prices and with exact input-token cost.
- Compliance-stratified quality frontiers.

### Technical Soundness Agent

Axis score: 4/10
Accept/reject signal: weak reject
Confidence: medium

#### Sound parts

- Expanding the routing search space is a valid way to create candidate operating points that point routers cannot represent.
- The paper correctly notices that response quality can vary with output length.

#### Soundness concerns

- The optimization theorem is a weak set-inclusion statement and does not validate the learned predictor, the cost frontier, or realized routes.
- MSE-trained quality predictors are used for argmax routing; without uncertainty or ranking calibration, this can create decision bias.
- The Qwen embedding model, Qwen judge, and Qwen-family routed models create possible family/preference bias that is not fully isolated.

#### Claim-support audit

- Claim: model-budget routing is a useful paradigm.
  Support: supported.
  Verdict: supported.
- Claim: R2-Router achieves reliable 4-5x lower cost than existing routers.
  Support: incomplete accounting, replay, and realization evidence.
  Verdict: partially supported to unsupported.

### Novelty and Positioning Agent

#### Claimed contribution

- Routing over model and output-length budget, plus R2-Bench.

#### Novelty-positive evidence

## Master Synthesis

R2-Router has a strong central idea: an LLM router should not treat each model as a single fixed quality-cost point when output length can be controlled. R2-Bench is also potentially useful because it records responses over multiple budgets. My score is below accept because the submission's strongest claim is the empirical 4-5x lower-cost frontier, and that claim is not yet sufficiently auditable. The paper says each response is annotated with actual token count and Appendix A studies compliance, but the public artifact does not include raw per-query traces or a full input/output cost manifest. The theorem is too weak to compensate: it proves only that a larger search space can weakly dominate a smaller one. Combined with possible MSE-to-argmax decision bias and Qwen-family entanglement between embedding/judge/models, the result remains promising but under-verified.
